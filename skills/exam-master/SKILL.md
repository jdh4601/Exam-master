# exam-master

## Overview

강의 자료를 분석하고, 강화학습 + 인지과학 원리를 적용한 8단계 시험 준비 루프를 실행하는 스킬.
사전 회상 → 개념 그래프 기반 출제 → 적응형 난이도 → 자기 설명 평가 → 간격 반복 복습의 사이클로 학습 효과를 극대화한다.

## 호출 방법

```
/exam-master {강의자료_경로}
```

예시:
```
/exam-master /Users/jayden/Documents/01_Class/데이터베이스/
/exam-master 데이터베이스/
```

경로가 없으면 AskUserQuestion으로 요청한다.

---

## Pre-flight: 자료 로드 및 복습 스케줄 확인

호출 시 가장 먼저 실행:

1. **경로 정규화**: 상대 경로는 절대 경로로 변환
2. **파일 수집**: `Glob("{경로}/**/*.md")` 로 모든 Markdown 파일 목록 수집
3. **파일 로드**: 각 파일을 `Read` 도구로 순서대로 읽기
4. **오류 처리**: 파일이 0개면 즉시 중단하고 경로 재확인 요청
5. **복습 스케줄 확인**: `error-notebook.md` 가 존재하면 `Read`로 로드하여 `next_review` 날짜가 오늘 이전인 항목을 추출 → 이번 세션 문제에 포함 예약

```
복습 예약 예시:
  🔁 복습 대상 개념: 정규화 (예정일: 2026-03-09, 3일 경과)
  🔁 복습 대상 개념: 트랜잭션 격리 수준 (예정일: 2026-03-08, 4일 경과)
```

---

## Pre-flight 이후 — 문제 수 선택

파일 로드와 복습 스케줄 확인이 끝나면, STEP 0 전에 문제 수를 묻는다:

사용자에게 출력:
```
몇 문제를 풀겠습니까?
① 5문제  ② 10문제 (기본)  ③ 15문제  ④ 직접 입력
(Enter 또는 '10' → 10문제)
```

선택된 문제 수(N)에 따라 레벨별 분배:

| 문제 수 | L1 | L2 | L3 | L4 | L5 |
|---------|----|----|----|----|-----|
| 5       | 1  | 1  | 1  | 1  | 1   |
| 10      | 1  | 2  | 3  | 2  | 2   |
| 15      | 2  | 4  | 4  | 3  | 2   |
| 직접 입력 | N을 기준으로 비율 유지하여 반올림 배분 |

이후 STEP 4에서 "{N}개를 풀겠습니다" 표시 및 문제 번호를 "문제 K/N"으로 출력.

---

## STEP 0 — 사전 회상 (Prior Recall)

**강의 자료를 분석하기 전에** 먼저 사용자의 기존 지식을 확인한다.

사용자에게 요청:
```
이번 과목에서 중요하다고 생각하는 개념을 3–5가지 적어주세요.
정확하지 않아도 됩니다. 지금 떠오르는 것을 자유롭게 써주세요.
```

사용자 답변 수신 후 간략 평가:

```
## 사전 지식 평가

✅ 정확한 이해: [개념명] — 이유
⚠️ 오개념 가능성: [개념명] — 어떤 점이 잘못됐는지 한 줄 설명
❌ 누락된 핵심 개념: [개념명] — 이 개념이 왜 중요한지 한 줄 힌트
```

이 평가 결과는 STEP 4 출제 시 오개념 개념의 가중치를 높이는 데 활용한다.

---

## STEP 1 — 핵심 개념 추출

로드한 자료 전체를 분석하여:

- **반복 등장 개념**: 여러 파일에 걸쳐 나타나는 개념
- **강조된 개념**: 굵은 글씨, 별도 섹션, 예시 설명이 많은 개념
- **예제/응용 설명**: 문제 풀이나 시나리오로 설명된 개념

출력 형식:
```
## 핵심 개념 맵

### [개념 A]
- 정의: ...
- 관련 개념: B (비교), C (선수 지식), D (인과)
- 강조 수준: 높음/중간/낮음
- STEP 0 오개념 플래그: 있음/없음
```

---

## STEP 2 — 지식 구조 재구성 (개념 그래프)

계층 구조로 정리하되, 개념 간 **관계 유형**을 명시한다:

```
{과목명}
├── 대주제 1
│   ├── 소주제 1-1
│   │   └── 핵심 개념: A, B
│   └── 소주제 1-2
│       └── 핵심 개념: C
└── 대주제 2
    └── ...
```

관계 유형:
- **인과** (`→`): "A이기 때문에 B가 발생"
- **선수관계** (`requires`): "C를 이해해야 D 이해 가능"
- **비교/대조** (`vs`): "E와 F의 차이점은..."
- **포함** (`⊂`): "G는 H의 특수 케이스"

이 개념 그래프는 STEP 4의 문제 생성 기반으로 사용된다.

---

## STEP 3 — 출제 스타일 예측

강의 자료 기반으로 예상 문제 유형 5가지를 추론하고, 각 유형이 출제될 이유를 강의 근거와 함께 설명:

```
예상 유형 1: 개념 정의 → 근거: ○○ 개념이 3회 이상 반복 등장
예상 유형 2: 비교 설명 → 근거: A와 B가 항상 대조되어 설명됨
예상 유형 3: 오개념 교정 → 근거: STEP 0에서 오개념 발견됨
...
```

---

## STEP 4 — 시험 문제 생성 (개념 그래프 기반 + 적응형 난이도)

### 난이도 체계 (5단계)

| 레벨 | 유형 | 설명 |
|------|------|------|
| Level 1 | 암기 | 정의, 이름 답하기 |
| Level 2 | 설명 | why/how 서술 |
| Level 3 | 복합 추론 | 2개 이상 개념 연결 |
| Level 4 | 적용 | 시나리오 설계·응용 |
| Level 5 | 오개념/함정 | 흔한 오해를 검증하는 역질문 |

### 초기 세션 기본 구성

| 레벨 | 문제 수 | 비고 |
|------|---------|------|
| Level 1 | 2문제 | — |
| Level 2 | 4문제 | — |
| Level 3 | 4문제 | — |
| Level 4 | 3문제 | — |
| Level 5 | 2문제 | STEP 0 오개념 우선 출제 |

**복습 대상이 있을 경우**: Pre-flight에서 추출한 복습 개념을 Level 3–5 문제 중 최대 3문제에 반영한다.

### 개념 그래프 기반 출제 원칙

단순 암기 문제보다 **개념 간 관계**를 묻는 문제를 우선한다:

```
인과 관계 → "A가 발생하면 B에 어떤 영향이 있는가?"
선수 관계 → "C 없이 D를 구현할 수 없는 이유는?"
비교/대조 → "E와 F를 선택 기준과 함께 비교하라"
오개념    → "다음 설명이 틀린 이유를 설명하라: ..."
```

### 적응형 난이도 조정 (2회차 이후)

이전 세션 총점 기준:
- **점수 ≥ 80%** → 다음 세션에서 Level 1 비중 줄이고 Level 4–5 비중 증가
- **점수 ≤ 50%** → Level 3–5 비중 줄이고 Level 1–2 비중 증가
- **50% < 점수 < 80%** → 현재 구성 유지

현재 난이도 구성을 첫 문제 출력 전에 한 번만 명시:
```
📊 이번 세션 난이도: L1×2 / L2×4 / L3×4 / L4×3 / L5×2
   (이전 세션 점수 기반 조정됨)
```

### 순차 출제 규칙 (CRITICAL)

**문제는 반드시 한 번에 1개씩 출제한다.** N개를 한꺼번에 출력하지 않는다.

출제 → 답변 대기 → 즉시 채점 → 다음 문제 출제 순서로 진행한다.

각 문제 출력 형식:
```
---
📌 문제 {K}/{N}  [Level {레벨}]

{문제 내용}

답변 전에 먼저 "이렇게 생각한 이유"를 한두 줄 써주세요.
형식: [이유] → [답변]
---
```

사용자 답변을 받으면 **즉시 STEP 5 채점**을 수행하고, 바로 다음 문제를 출제한다.
N번째 문제 채점이 끝나면 전체 총점 요약을 출력한다.

### '종료' 입력 처리

사용자가 문제에 답하는 대신 **'종료'** 를 입력하면:
1. 현재 문제 출제를 즉시 중단
2. "문제 출제를 종료합니다. 지금까지 K문제 중 채점된 결과로 마무리합니다." 출력
3. 채점된 문제들의 점수만 합산하여 총점 계산 (예: 7문제 채점 → X/70)
4. 바로 STEP 6(오답 노트 저장)으로 이동

단, 아직 채점되지 않은 현재 문제는 채점 기록에서 제외한다.

### 규칙

- 답을 절대 보여주지 않는다 (채점 전까지)
- 단순 복사·붙여넣기로 맞출 수 없는 문제만 출제
- 강의 자료에 없는 내용은 출제 금지
- 문제를 출력한 뒤 반드시 멈추고 사용자 답변을 기다린다

---

## STEP 5 — 자기 설명 평가 + 채점 (문제별 즉시 채점)

사용자가 답변하면 **그 문제에 대해 즉시** 채점 결과를 출력한다.

각 문제 채점 출력 형식:
```
### 문제 {N} 채점 — [문제 핵심 키워드]
- 평가: ⭕ 정답 / 🔺 부분 정답 / ❌ 오답
- 점수: X/10
- 추론 평가: 논리 흐름이 맞음 / 결론은 맞지만 추론이 잘못됨 / 추론과 결론 모두 틀림
- 정확한 내용: ...
- 누락된 내용: ...
- 강의 근거: "{파일명}, {섹션명}" 참조
```

채점 직후 다음 문제를 바로 출제한다. (별도 확인 없이 자동 진행)

> **추론이 틀렸지만 결론이 맞은 경우**: 부분 정답(🔺)으로 처리하고 오답 노트에 포함한다.
> 올바른 결론을 잘못된 방법으로 도달한 것도 교정이 필요하기 때문이다.

**N번째 문제 채점 후** (또는 '종료' 입력 후) 전체 세션 요약을 출력한다:

```
## 📊 세션 종료 요약

### 총점
  - 결과: XX / {만점} ({퍼센트}%)
  - 문제 구성: L1×N / L2×N / ...

### 🎯 지금 당장 복습할 개념 (우선순위 순)
  오답·부분정답 중 "처음 틀린 개념"을 우선순위 높게 나열:
  1. [개념명] — 틀린 이유 한 줄 요약
  2. [개념명] — ...
  (복습 노트 저장 후 내일 다시 출제 예정)

### 📡 학습 레이더 차트
  강의 자료에서 추출한 주요 개념 영역별 점수를 텍스트 바 차트로 시각화.
  각 영역의 점수 = 해당 영역에서 출제된 문제들의 평균 점수 / 10 × 100%

  형식 예시:
  ┌─────────────────────────────────┐
  │ 데이터 기초          ████████░░ 80% │
  │ 파일시스템/DBMS      ██████░░░░ 60% │
  │ 3단계구조/독립성     ██░░░░░░░░ 20% │
  │ 데이터 언어          █████░░░░░ 50% │
  │ DBMS 발전사          ███████░░░ 70% │
  └─────────────────────────────────┘
  ■ = 달성, ░ = 미달성 (10칸 = 100%)

  영역 분류는 STEP 2 개념 그래프의 대주제를 기반으로 자동 설정.
  해당 영역 문제가 0개인 경우 "—" 표시.

### 📈 성장 추이 (2회차 이후)
  이전 세션과 비교하여 영역별 변화 표시:
  예) 데이터 기초: 60% → 80% (+20%) ↑

### 🎯 다음 세션 난이도 조정
  [조정 방향 및 이유]
```

## STEP 6 — 오답 노트 생성 및 저장 (간격 반복 스케줄 포함)

오답(❌)과 부분 정답(🔺) 문제만 선별하여 Error Note 작성.

**저장 경로**: `{자료경로}/error-notebook.md`

### 저장 로직

```
error-notebook.md 존재 여부 확인 (Read 시도):
  - 없음 → Write 도구로 새 파일 생성
  - 있음 → Edit 도구로 파일 끝에 오늘 세션 내용 추가
           (기존 항목의 next_review가 갱신된 경우 해당 항목도 Edit으로 업데이트)
```

### 파일 형식

```markdown
# 오답 노트 — {과목명}
> 마지막 업데이트: {YYYY-MM-DD}

---

## 세션 {YYYY-MM-DD} — {총점}/{만점} | 난이도: L1×N/L2×N/...

### 개념: {개념명}
- **출제 문제**: ...
- **내 답변 요약**: ...
- **추론 오류**: (추론 과정에서 어떤 점이 잘못됐는지)
- **실수 원인**: ...
- **정답 해설**: ...
- **기억 포인트**: (한 줄 요약)
- **next_review**: {YYYY-MM-DD}   <!-- 오늘 + 간격 -->
```

### 간격 반복 스케줄 규칙

| 조건 | 다음 복습 간격 |
|------|--------------|
| 처음 틀린 개념 | 1일 후 (`next_review: +1d`) |
| 두 번째 연속 오답 | 3일 후 (`next_review: +3d`) |
| 세 번째 이상 연속 오답 | 7일 후 (`next_review: +7d`) |
| 재출제(STEP 7)에서 정답 | 7일 후 (`next_review: +7d`) |

`next_review` 날짜는 오늘 날짜 기준으로 계산하여 실제 날짜(YYYY-MM-DD)로 기록한다.

---

## STEP 7 — 강화 재출제 (인터리빙 + 간격 반복)

STEP 6 오답 개념만 단독으로 반복하지 않는다. **인터리빙** 방식으로 구성:

### 5문제 구성 원칙

| 문제 수 | 출처 |
|---------|------|
| 2–3문제 | 이번 세션 오답 개념 (다른 각도로 재출제) |
| 1–2문제 | 오답 개념과 관련된 **다른 과목 개념** (개념 그래프의 인과·비교 관계 활용) |
| 1문제 | Pre-flight 복습 대상 중 아직 정답 처리 안 된 개념 (있을 경우) |

### 규칙

- 동일 문제 재사용 금지 — 같은 개념을 다른 각도로 새 문제 출제
- 주로 Level 2–4 수준으로 출제 (단순 암기 지양)
- 채점 후 다시 오답이 있으면 루프 반복 여부 확인:

```
STEP 5–7을 한 번 더 반복할까요?
취약 개념이 남아있습니다: [개념명], [개념명]
```

---

## 공통 규칙

- 모든 출력은 **한국어**로
- 강의 자료에 없는 내용은 절대 출제하지 않는다
- 평가·해설은 반드시 강의 파일 근거를 명시한다
- 한 STEP이 끝나면 다음 STEP으로 자동 진행
- **사용자 입력 대기**: STEP 0 (사전 회상), STEP 4 (답변 제출), STEP 5 이후 STEP 7 (루프 여부)

---

## 전체 실행 흐름 요약

```
Pre-flight  자료 로드 + 복습 스케줄 확인 (next_review 추출)
    ↓
문제 수 선택  5 / 10(기본) / 15 / 직접 입력 — 사용자 입력 대기
    ↓
STEP 0      사전 회상 — 사용자 입력 대기
    ↓
STEP 1      핵심 개념 추출 (오개념 플래그 포함)
    ↓
STEP 2      개념 그래프 구성 (관계 유형 명시)
    ↓
STEP 3      출제 유형 예측
    ↓
STEP 4      N문제 생성 (그래프 기반 + 적응형 난이도 + 복습 포함) — 사용자 입력 대기
            ('종료' 입력 시 채점된 결과로 즉시 STEP 6 이동)
    ↓
STEP 5      자기 설명 + 채점 → 마지막 문제 후 강화 요약 출력
            (총점 / 즉시 복습 목록 / 레이더 차트 / 성장 추이)
    ↓
STEP 6      오답 노트 저장 (next_review 날짜 기록)
    ↓
STEP 7      인터리빙 재출제 5문제 → 루프 여부 확인
```

---

## 흔한 실수

| 실수 | 올바른 방법 |
|------|------------|
| STEP 4에서 답 보여줌 | STEP 5 전까지 절대 답 미출력 |
| 강의에 없는 내용 출제 | 자료 근거 없으면 출제 금지 |
| error-notebook.md 덮어쓰기 | 기존 파일 있으면 Edit으로 누적 추가 |
| STEP 7에서 동일 문제 재사용 | 같은 개념을 다른 각도로 새 문제 출제 |
| 경로 인자 없이 시작 | AskUserQuestion으로 경로 요청 |
| 추론이 잘못됐는데 결론이 맞으면 정답 처리 | 🔺 부분 정답으로 처리하고 오답 노트에 포함 |
| STEP 7을 오답 개념만으로 구성 | 인터리빙: 오답 + 관련 개념 + 복습 대상 혼합 |
| next_review 날짜를 고정 간격으로만 계산 | 연속 오답 횟수에 따라 간격 조정 (1d/3d/7d) |
| 15개 문제를 한꺼번에 출력함 | 문제는 1개씩 출제 → 답변 대기 → 즉시 채점 → 다음 문제 순서로 진행 |
| 채점 후 다음 문제 출제를 사용자에게 물어봄 | 채점 직후 자동으로 다음 문제 출제 (별도 확인 불필요) |
| 문제 수를 물어보지 않고 바로 시작 | Pre-flight 직후 반드시 문제 수 선택 단계 실행 |
| '종료' 입력 후 세션 강제 종료 | 채점된 결과로 정상 STEP 6~7 흐름 마무리 |
| 레이더 차트 영역을 임의로 설정 | STEP 2 개념 그래프의 대주제 기반으로 설정 |
